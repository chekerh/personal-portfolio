## CV purpose
The CV must function as a high-speed credibility document. It should make recruiters and hiring managers understand the brand quickly, trust the project selection, and see a pattern of ownership rather than scattered activity.

## Primary outcome
Present Cheker Hassan as a product-minded software engineer and operator-builder with unusually strong range in:
- structuring products
- supervising delivery
- building ambitious systems
- connecting technical work to launch readiness

## Positioning rules
- Do not use generic “passionate software engineer” wording.
- Do not lead with student status.
- Do not flatten the CV into a tool list.
- Do use strong scope language when defensible:
  - structured
  - led
  - built
  - supervised
  - coordinated
  - delivered
- Separate hands-on implementation from product structuring or supervision when those roles differ.

## Content priorities
1. Summary that explains identity and target
2. Selected project-based experience with real systems
3. Delivery and leadership pattern
4. Skills tied to proof
5. Education and affiliations

## Default flagship projects for CV emphasis
- CheckAuto TN

## Variant strategy
### Variant A: General software engineering
- Strongest focus on systems built, stacks used, and product complexity

### Variant B: Product / delivery-heavy engineering
- Strongest focus on roadmaps, Jira planning, coordination, technical supervision, and launch framing

### Variant C: Greece-focused application version
- Strongest focus on clarity, breadth, initiative, languages, and professional readiness

## Bullet-writing rules
- Keep bullets outcome-oriented
- Prefer one clear idea per bullet
- Mention user or business context when helpful
- Mention technology only when it supports credibility
- Avoid inventing metrics; only use numbers that can be defended

## Relationship to other public surfaces
- Project names must match `naming-normalization.md`
- Featured projects must map to the same flagship/support hierarchy as the website
- GitHub pinned repos should reinforce the projects named in the CV
- LinkedIn experience entries should not contradict CV project framing
