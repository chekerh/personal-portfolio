## Pass objective
Create the foundation required to execute the premium personal brand program without ambiguity in scope, positioning, or priorities.

## Inputs
- Existing portfolio site in `/Users/mac/PersonalPortfolio/salimov`
- Current portfolio content in `salimov/src/content/portfolioData.js`
- Existing CV assets in repo root
- Existing repo inventory and portfolio notes in repo root markdown files

## Outputs
This pass must produce:
- program charter
- roadmap
- decision log
- initial positioning baseline
- project inventory framing rules
- implementation constraints for the website pass

## Scope
Included:
- define goals, audience, and success outcomes
- define the positioning direction
- define delivery phases
- define key decisions and constraints
- define what later passes must implement

Excluded:
- direct file edits to the website
- final CV rewrite
- final LinkedIn rewrite
- repo-by-repo GitHub cleanup
- visual QA and launch checks

## Working assumptions
- The current site is good enough structurally to adapt in the first implementation cycle
- The strongest personal brand angle is operator-builder with technical and delivery ownership
- A smaller featured-project set will outperform a broad repo dump
- Content consistency matters more than adding new sections

## Foundation deliverables
### 1. Positioning baseline
Define:
- core identity
- target audience
- target roles
- premium tone
- proof standard

### 2. Program controls
Define:
- scope
- milestones
- decision logging rules
- completion criteria

### 3. Website constraints
Define:
- keep current Next.js base unless blocked
- prioritize content and hierarchy before design flourishes
- use featured projects as the main proof block
- ensure resume and contact flows are obvious

### 4. Project inventory direction
Define:
- shortlist principle: fewer, stronger projects
- each featured project needs a clear angle, stack, and proof
- supporting projects can exist outside the homepage spotlight

## Tasks
- Draft charter with scope, goals, and done criteria
- Draft roadmap with milestone order and exit criteria
- Draft decision log with initial approved decisions
- Confirm the implementation target is the `salimov` site
- Establish what the next pass must create

## Exit criteria
Pass 01 is complete when:
- scope is explicit
- positioning is explicit
- milestones are explicit
- early decisions are documented
- the next implementation pass can start without reopening strategy questions

## Risks to carry forward
- the current site may still feel template-like without stronger visual differentiation
- some flagship claims may need stronger artifacts or screenshots
- project inventory may still be too broad until the shortlist is finalized
- CV variants may be needed later depending on application targets

## Next pass
Pass 02 should focus on content core creation:
- homepage copy
- featured-project summaries
- CV master bullets
- LinkedIn and GitHub alignment copy
