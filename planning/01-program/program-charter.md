## Program
Premium Personal Brand Program for Cheker Hassan

## Repository context

## Purpose
Build a premium, recruiter-facing personal brand system that presents Cheker Hassan as an operator-builder and product engineer with strong product delivery instincts, technical supervision signals, and credible launch-oriented execution across ambitious software products.

## Scope
This program covers:
- Personal website
- CV and resume package
- GitHub profile and repository presentation
- LinkedIn profile alignment
- Project inventory and featured case-study selection

This program does not cover:
- New product development unrelated to personal brand presentation
- Paid acquisition or ad operations
- Full long-form content marketing as a first milestone
- Deployment or publishing mechanics beyond launch readiness planning

## Primary goal
Create a coherent personal brand across all candidate-facing surfaces so that employers, founders, and collaborators quickly understand:
- who Cheker is
- what kind of systems he builds
- how he works
- why he is a strong fit for engineering, product-delivery, and operator-builder roles

## Success outcomes
- Website clearly communicates positioning, proof, and target roles in the first screen and first scroll
- CV, LinkedIn, GitHub, and website use the same project hierarchy and identity
- Featured projects are curated and described consistently
- The system supports applications to Greece-based and remote-first roles
- The content model is maintainable and reusable rather than rewritten per channel

## Target audience
- Engineering recruiters
- Hiring managers
- Founders and startup operators
- Technical leads evaluating junior-to-mid builders with ownership potential
- Collaborators reviewing product and execution ability

## Positioning statement
Cheker Hassan is a software builder and operator-builder who turns ambitious ideas into structured, launchable products through planning, technical supervision, and hands-on execution across SaaS, backend systems, developer tools, AI workflows, and operational software.

## Core deliverables
- Portfolio positioning and messaging baseline
- Upgraded website structure and content plan
- Project inventory with featured-project shortlist
- CV strategy and role-variant plan
- LinkedIn headline, About, experience, and Featured alignment
- GitHub profile and repository curation plan
- Execution passes with QA checkpoints

## Constraints
- Work within the current `salimov` Next.js site unless a rebuild is explicitly approved later
- Prioritize clarity, cohesion, and proof over large technical rewrites
- Use only claims that can be defended through repos, screenshots, docs, artifacts, or direct experience
- Keep the brand premium, specific, and credible rather than inflated

## Risks
- Inconsistent narrative across channels
- Too many projects diluting the strongest signals
- Weak proof for flagship claims if screenshots and docs are not curated
- Template-driven site structure limiting differentiation
- Spending time polishing visuals before fixing hierarchy and positioning

## Guiding decisions
- Narrative first, visuals second
- Fewer, stronger projects over exhaustive listing
- Premium clarity over generic “full-stack developer” messaging
- Proof-backed claims only
- One canonical content source reused across all channels

## Governance
- Major changes to positioning, featured projects, target market, or proof standards must be logged in `decision-log.md`
- Each execution pass should end with a visible deliverable and review note
- Root planning docs are the permanent memory layer for future agent-mode work

## Definition of done
The program is complete when:
- the website is portfolio-ready
- the CV is application-ready
- GitHub and LinkedIn are aligned
- featured projects are curated and documented
- and the brand system is consistent enough to support active applications without ad hoc rewrites
