## Purpose
This folder is the long-term memory layer for Cheker Hassan's personal brand program. It exists to prevent drift between the website, CV, GitHub, LinkedIn, and future outreach materials, and to make future agent-mode implementation possible without re-deriving strategy from scratch.

- CV strategy and role-targeted variants

## Conventions
- Every planning file starts with `Status`, `Owner`, `Last updated`, `Depends on`, and `Feeds into`.
- Internal repo names stay in planning docs for traceability.
- Public-facing product names are used on the site, CV, GitHub profile README, and LinkedIn.
- Claims should be phrased according to evidence:
  - `Confirmed`: supported by README, code structure, docs, or visible artifact
  - `Partially confirmed`: supported by naming, architecture files, or adjacent notes
  - `Narrative-only`: plausible framing that still needs direct proof before public use
- `decision-log.md` is the single place where final strategic decisions are recorded.

## Canonical truth rules
- `03-content/master-profile.md` is the source of truth for identity, contact, education, goals, and recurring positioning facts.
- `03-content/project-inventory.md` is the source of truth for what exists.
- `03-content/featured-projects.md` is the source of truth for flagship/support/hidden public presentation decisions.
- `04-site/*`, `05-cv/*`, and `06-profiles/*` must not invent facts that conflict with `03-content/*`.
